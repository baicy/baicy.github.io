CDN与智能DNS原理和应用

发表于 2019-05-01

可能一直都不太清楚这方面的知识吧 (:з」∠)

学习视频

这方面的知识，对应的工作技能是网站速度优化

CDN:Content Delivery Network 内容分发网络

使用户可以就近取得所需内容，提高用户访问网站的响应速度

CDN=更智能的镜像+缓存+流量导流

DNS:Domain Name System，域名系统

域名和IP地址相互映射的一个分布式数据库，不用去记住能够被机器直接读取的IP

CDN加速原理：内容缓存(CDN的本质)，分布式多节点(主要加速原理)，多线路支持(主要减少跨网访问)

CDN适用范围：静态和更新频率低的内容，数据流量大的产品，带宽价格更便宜

CDN具体使用：管理(缓存策略，手动刷新)，统计(命中率，用户来源)

DNS解析过程：
客户机发起url请求 –> 在本地DNS解析缓存里面查找对应的ip,找到则返回，找不到 –> 在本地hosts文件查找，找到返回，否则 –> 在本地DNS服务器查找，找到返回，否则 –> 在区域内DNS服务器查找，同上 –> 在更大范围的DNS服务器查找，同上 –> 根级DNS服务器上查找，到这里之后无论能找到还是不能找到，都将结束。

过程比较繁琐，如果都没有缓存，有时候确实会比较慢

智能DNS：根据内置的丰富IP库来为用户匹配合适的线路（移动、联通、电信…）以及识别用户位置提供更近的网络服务。

总结：好的CDN服务让互联网产品更简单高效和可靠

  • 加快访问速度

  • 缓解源服务器压力

  • 让大规模的用户请求架构变简单

  • 降低流量成本

最后说到物流系统很像CDN，这个点我竟然感觉也是个新知识
