Vue项目发布在github预览

发表于 2019-09-05

前端项目想发布在网上供大众预览有很多种方法，比如将要使用的利用gh-pages发布项目

使用的Vue.js版本是3.11.0，版本不同会有些微的文件结构不同

以自己的项目arkObserver为例，预览地址

编译项目

在项目根目录新建文件vue.config.js，将publicPath更改为/项目名/

默认路径是空，gh-pages会自动使用xxx.github.io/项目名/

在项目根目录运行npm run build打包项目

在项目根目录生成了dist文件夹

上传github

有2种情况

1.项目代码同时发布在github上，利用branch gh-pages预览

2.不发布项目代码，只部署branch master预览

本例属于情况1，已有github respository

gh-pages配置

在github项目仓库选择Settings-Options-GitHub Pages

Source处选择gh-pages branch，稍等一会，会显示Your site is published at http://xxx

部署完成

CDN与智能DNS原理和应用

发表于 2019-05-01 | 更新于 2019-05-02

可能一直都不太清楚这方面的知识吧 _(:з」∠)_

学习视频

这方面的知识，对应的工作技能是网站速度优化

CDN:Content Delivery Network 内容分发网络

使用户可以就近取得所需内容，提高用户访问网站的响应速度

CDN=更智能的镜像+缓存+流量导流

DNS:Domain Name System，域名系统

域名和IP地址相互映射的一个分布式数据库，不用去记住能够被机器直接读取的IP

CDN加速原理：内容缓存(CDN的本质)，分布式多节点(主要加速原理)，多线路支持(主要减少跨网访问)

CDN适用范围：静态和更新频率低的内容，数据流量大的产品，带宽价格更便宜

CDN具体使用：管理(缓存策略，手动刷新)，统计(命中率，用户来源)

DNS解析过程：
客户机发起url请求 –> 在本地DNS解析缓存里面查找对应的ip,找到则返回，找不到 –> 在本地hosts文件查找，找到返回，否则 –> 在本地DNS服务器查找，找到返回，否则 –> 在区域内DNS服务器查找，同上 –> 在更大范围的DNS服务器查找，同上 –> 根级DNS服务器上查找，到这里之后无论能找到还是不能找到，都将结束。

过程比较繁琐，如果都没有缓存，有时候确实会比较慢

智能DNS：根据内置的丰富IP库来为用户匹配合适的线路（移动、联通、电信…）以及识别用户位置提供更近的网络服务。

总结：好的CDN服务让互联网产品更简单高效和可靠

  • 加快访问速度

  • 缓解源服务器压力

  • 让大规模的用户请求架构变简单

  • 降低流量成本

最后说到物流系统很像CDN，这个点我竟然感觉也是个新知识

孩子，多学习，多思考吧
