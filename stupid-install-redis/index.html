不熟练总会出点事，记安装配置php+redis

发表于 2019-05-02

虽然想做点项目唤醒一下自己的兴趣和熟练度，但是深深的发现最基础的事情都还madamada。

之前配置laraval的时候折腾了一下Homestead和Valet，事到如今好像也不是很记忆深刻。

简单来说就是还是要给自己的环境配置一下redis，还是要熟练一下过程。

以及docker的使用应该提上日程了。

安装环境是mac OS 10.14.4，终端下查看mac版本使用sw_vers

首先是安装redis，本来是准备用源码安装的……想了想还是直接用brew了

安装完后修改一下redis配置

brew info redis可以查看redis.conf的位置

修改完后用brew启动redis，会按照更改后的配置

非brew启动使用 redis-server .../redis.conf一般会在后面带上配置文件

确认redis server已经启动

接下来终端访问以下redis客户端，这时候直接访问会被拒绝，因为默认是用6379端口登录

登陆后info查看redis信息，可以确认使用的配置文件

接下来安装PHP的Redis扩展

一开始也是源码编译路线……(这句话后面有提及)

首先确认PHP版本，虽然对于这个差别不大php -v

php -m确认机器上是否已经装了redis扩展，好的并没有

之后用which看看机器上是否安装了phpize和php-config，记住php-config的安装路径

使用wget下载redis扩展，发现mac并没有预装wget，总之用brew安装了wget，然后下载了delevop.zip

此处做个笔记，关于wget和curl
wget是unix上一个发送网络请求的命令工具，不过mac本身并没有，mac自带的是curl，都是发送网络请求，但是两者之间肯定存在一些差异。一般来说，wget主要专注于下载文件，curl长项在于web交互、调试网页等。
具体分析参考curl和wget的区别
